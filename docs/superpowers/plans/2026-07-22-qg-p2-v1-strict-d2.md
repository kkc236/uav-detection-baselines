# QG-P2 v1 Strict D2 Implementation Plan

**Goal:** Implement the frozen QG-P2 quality gate, verify isolation and query
integrity, create a derived immutable initial state, and run a fresh strict A0/QG
D2 pair before considering 100 epochs.

**Architecture:** Add a detached-input scalar quality head and sparse IoU quality
loss to the existing P2 side branch. Use joint class-quality probability only for
P2 ranking. Preserve the stock path, P2 sparse loss, fixed query budget, training
protocol, and validation implementation.

## Task 1: Freeze Design And Evidence

- [x] Record the four-arm trigger metrics and mechanism evidence.
- [x] Freeze the one-variable quality-head target, loss, ranking, initialization,
  activation schedule, and gates.
- [ ] Hash and publish the completed A1-no-injection evidence set.

## Task 2: Quality Target And Loss TDD

- [ ] Add failing tests for sparse scalar targets, detached IoU targets, excluded
  non-tiny negatives, finite empty batches, and quality-head-only gradients.
- [ ] Implement the minimal quality loss without changing existing P2 loss.
- [ ] Run focused loss tests.

## Task 3: Decoder And CLI TDD

- [ ] Add failing tests for zero initialization, joint ranking, stock-only warmup,

  `qg-p2` D2 arm with `lambda_ebc=0` and fusion gamma enabled.
- [ ] Prove one-batch finite nonzero quality-head gradients and no quality-loss

## Task 5: Local And Server Verification

## Task 6: Fresh Strict D2 Pair

- [ ] Derive and freeze the QG initial state and protocol manifest from the frozen
  parent state; verify unchanged common and existing innovation fingerprints.

- [ ] Run QG-P2 for 10 epochs under the identical protocol with separate per-epoch
  upload.
- [ ] Revalidate both final checkpoints with exact metrics and run read-only QG

## Task 7: Gate And Freeze
