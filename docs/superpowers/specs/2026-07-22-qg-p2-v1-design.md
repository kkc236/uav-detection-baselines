# QG-P2 v1 Strict D2 Design

## Objective

Preserve the metric benefit of the P2 training branch while replacing the harmful
raw-score query injection with a conservative learned quality gate. This is a new
frozen version and does not reuse a training trajectory from EBC-QP v1.

## Triggering Evidence

The frozen four-arm seed-0 D2 report records:

- A1-no-injection versus A0: six wins and one tie over active epochs, final-three
  mean mAP50-95 delta `+0.0001866667`, and exact mAP50-95 delta
  `+0.0004329123`.
- A1 versus A1-no-injection: one win and six losses, final-three mean mAP50-95
  delta `-0.0001866667`, and exact mAP50-95 delta `-0.0004367942`.
- Raw query injection: `N_gain=707`, `N_loss=938`, `V_replace=-231`, P2 Top-50
  background rate `0.9621898`, unassigned entry rate `0.9932482`, and score/IoU
  Spearman `0.0597518`.

The P2 training signal therefore passes its metric gate, while raw-score query
injection fails both the metric and mechanism gates. Duplication is not the
primary failure, so local-peak filtering is not selected.

## Frozen QG-P2 Change

QG-P2 adds one scalar quality head to each P2 token:

```text
quality_logit = Linear(stop_gradient(P2 transformed feature))
```

The linear weight and bias are initialized to zero, so the initial quality
probability is exactly `0.5`. The detached input ensures the quality objective
does not alter the P2 adapter, stock encoder transform, stock score head, or P2
box head. The existing P2 sparse loss remains unchanged.

For the same sparse classification support used by P2 VFL:

- a locally assigned tiny-GT candidate receives detached predicted-box IoU as
  its scalar quality target;
- an eligible unassigned Top-50 candidate receives target zero;
- candidates whose centers lie inside a non-tiny GT remain excluded from the
  negative set, exactly as in the existing P2 classification loss.

Quality loss is binary cross entropy with logits, normalized by the number of
positive local assignments (minimum one):

```text
L_total = L_stock + 0.25 * L_P2 + 0.25 * L_QG
```

EBC is disabled in QG-P2 v1. No other auxiliary loss or parameter is added.

## Quality-Gated Ranking

Stock ranking remains its original maximum class logit. P2 ranking uses a joint
probability and converts it back to a logit so it remains comparable with stock:

```text
p_class = sigmoid(max_class_logit)
p_quality = sigmoid(quality_logit)
p_joint = clamp(p_class * p_quality, eps, 1-eps)
p2_ranking_logit = logit(p_joint)
```

P2 Top-50 selection and stock/P2 global Top-300 competition use this ranking.
There is no fixed P2 quota, hard threshold, local peak filter, NMS, or dynamic
query count. Ties continue to prefer stock.

Epochs 1-3 train P2 and QG losses but keep all final decoder queries stock.
Quality-gated competition activates at epoch 4.

## Initialization And Pairing

Create a new immutable QG initial-state artifact by copying the exact frozen
common state and existing P2/gamma innovation tensors, then adding only the
zero-initialized quality-head tensors. Record the parent artifact SHA256 and new
innovation fingerprint. The strict QG control loads only the unchanged common
state; the method loads common plus all QG innovation tensors.

Run a fresh A0 control and QG-P2 method on the same RTX 4090, seed 0, fixed D2
subset, sample order, augmentation stream, validation code, and complete common
training protocol. Existing A0 results are evidence but are not substituted for
the fresh QG control.

## Gates

QG-P2 may proceed beyond D2 only when both gates pass:

1. Metric gate: the frozen active-epoch trajectory rule passes against the fresh
   control, final-three mean mAP50-95 is non-negative, and exact tiny recall
   improves.
2. Mechanism gate: `N_gain > N_loss` and `V_replace > 0` on full validation.

Secondary evidence must show positive P2 entry count without Top-50 saturation,
and quality ranking should improve background/unassigned rates or score-quality
correlation relative to frozen raw injection. Secondary diagnostics cannot
override either primary gate.

No 100-epoch run starts unless both primary gates pass. Failure creates an
immutable negative result; parameters or thresholds are not edited in-place.
