# 无人机目标检测研究项目总览

更新日期：2026-07-21

## 1. 项目目标

本项目以 RT-DETR-L 为基础，在 VisDrone 无人机航拍数据集上研究复杂场景目标检测。最终目标不是简单拼接已有模块，而是围绕无人机检测中的三个科学问题，提出可解释、可复现且具有正向实验收益的方法，并形成一篇 C 类会议论文。

三个科学问题为：

1. 复杂背景中的小目标响应弱，容易被道路、建筑和纹理干扰。
2. 车辆、行人等目标分布密集，相邻实例容易发生查询竞争和边界混淆。
3. 飞行高度、俯视角和透视变化导致目标表观尺度不稳定。

项目最终成功标准包括：

- 在严格匹配的 RT-DETR-L baseline 上取得可重复的正向增益。
- 主要指标使用 mAP50-95，同时报告 Precision、Recall、mAP50 和针对性指标。
- 模块机制、实验现象和论文表述一致。
- 参数量、GFLOPs 和推理延迟满足轻量化约束。
- 代码、配置、日志、指标和权重能够在新服务器上恢复和复现。

## 2. 前期完成的工作

### 2.1 文献与问题梳理

项目从 YOLO、YOLOv3、Faster R-CNN、RT-DETR、DEIM-DETR、FBRT-YOLO 和 UAV-DETR 等工作入手，完成了目标检测主要技术路线的梳理。

得到的基本认识是：

- YOLO 路线强调单阶段实时检测，但对小目标、密集目标和复杂背景仍需针对性增强。
- Faster R-CNN 代表两阶段精度路线，但实时部署成本较高。
- RT-DETR 提供端到端、无 NMS 的实时检测基础，适合作为三个创新模块的统一载体。
- 现有无人机检测工作大多属于特征融合、注意力、多尺度或损失改进，因此本项目必须通过清晰的问题定义、独立机制和消融实验建立创新边界。

### 2.2 数据与基础模型

当前实验使用：

- 数据集：VisDrone train/val。
- 训练集：6471 张图像。
- 验证集：548 张图像。
- 类别数：10。
- 基础模型：Ultralytics RT-DETR-L。
- 初始化：从零训练，`pretrained=False`。
- 输入分辨率：640。
- 主要硬件：RTX 4090 24GB。

数据集在不同服务器上的标签和图像清单签名已经核对一致，避免了不同服务器实际使用不同数据版本的问题。

## 3. 三个创新点

### 3.1 创新点1：BTD-SE

正式名称：背景参照引导的目标-背景解耦与显著性增强模块。

英文名称：Background-Reference-Guided Target-Background Decoupling and Saliency Enhancement Module。

目标：解决复杂背景下小目标响应弱、容易被道路和建筑纹理淹没的问题。

核心机制：

1. 根据浅层细节特征和高层语义预测背景可靠性图。
2. 在中心排除的环形邻域内，对可靠背景进行动态加权聚合。
3. 使用当前特征与背景参照的残差表示目标偏离响应。
4. 通过语义一致性确认残差，并生成独立显著性图。
5. 使用通道缩放参数将残差分支注入 RT-DETR 特征路径。

主要监督包括背景有效性损失和显著性损失。检测主路径使用 AMP，数值敏感的辅助 Focal Loss 使用 FP32。

当前实现的额外开销较小：参数量约增加 0.053%，GFLOPs 约增加 0.28%。

### 3.2 创新点2：IOQC-SA

正式定位：实例归属感知的查询竞争与尺度对齐训练方法。

英文名称：Instance-Ownership-Aware Query Competition and Scale Alignment。

目标：解决密集目标场景中多个查询竞争同一实例、相邻实例容易混淆的问题。

核心机制：

1. 在训练阶段读取 RT-DETR 最后一层解码器的 P3 采样统计。
2. 使用原始 Hungarian 匹配确定每个 GT 的 owner query。
3. 从未匹配查询中选择同一目标的 Top-1 duplicate query。
4. 对 duplicate 施加单向竞争约束，避免多个查询聚集到同一实例。
5. 对 owner query 施加中心和尺度对齐约束。

IOQC-SA 是训练期方法，不改变正式推理图，因此理论上不增加推理参数量和推理延迟。

### 3.3 创新点3：VSF-RMR

正式名称：视角尺度场引导的残差式多尺度路由模块。

英文名称：View-Scale-Field-Guided Residual Multi-Scale Routing。

目标：解决无人机高度、俯视角和透视变化导致目标尺度不稳定的问题。

核心机制：

1. 从 P3、P4 和 P5 特征估计连续表观尺度场。
2. 将连续尺度映射为相邻层级的有序路由权重。
3. 构造跨尺度路由残差，而不是直接替换原始特征。
4. 使用零初始化通道缩放保持训练初始阶段接近原 RT-DETR 路径。
5. 使用由 GT 尺寸生成的局部和全局尺度监督训练尺度场。

当前实现不依赖无人机高度、姿态或焦距元数据，但在尺度 logit 位置保留未来元数据扩展接口。

## 6. 当前实验结果

所有数值均使用各实验在统一 fitness 规则下的最佳轮次。新 matched baseline 的精确 CSV 仍需统一归档，因此当前从完整日志读取到三位小数。

| 模型 | 最佳 Epoch | Precision | Recall | mAP50 | mAP50-95 |
| --- | ---: | ---: | ---: | ---: | ---: |
| Matched RT-DETR-L baseline | 100 | 约 0.511 | 约 0.435 | 约 0.415 | 约 0.241-0.242 |
| BTD-SE | 66 | 0.44098 | 0.38683 | 0.35068 | 0.19867 |
| IOQC-SA | 100 | 0.51443 | 0.42183 | 0.40575 | 0.23335 |
| VSF-RMR | 69 | 0.45061 | 0.40352 | 0.35943 | 0.20374 |

基于当前单次 seed-0 结果：

- BTD-SE 比新 baseline 低约 4.3 AP。
- IOQC-SA 比新 baseline 低约 0.8 AP，但 Precision 略高、Recall 较低。
- VSF-RMR 比新 baseline 低约 3.8 AP。

因此，三个模块当前都不能在论文中宣称总体指标超过 matched baseline。

## 7. 当前结果说明了什么

### 7.1 已经成立的事实

- 三个模块均已实现并能够端到端训练。
- 三个模块的独立实验边界已经建立，没有互相混入代码。
- IOQC-SA 最接近 matched baseline，并表现出提高 Precision、降低 Recall 的明确倾向。
- BTD-SE 的背景可靠性、显著性和残差缩放分支均产生了有效梯度。
- VSF-RMR 的尺度场与 GT 尺寸具有较高相关性，路由权重不是常数。
- 服务器自动恢复和 GitHub 保护系统已经能支撑长时间训练。

### 7.2 尚未成立的结论

- 不能声称 BTD-SE 已经提高复杂背景小目标 AP。
- 不能声称 IOQC-SA 已经提高整体检测 AP。
- 不能声称 VSF-RMR 已经提高尺度变化鲁棒性。
- 不能把旧 5090 adaptive baseline 与固定协议创新实验直接比较。
- 不能只根据辅助损失下降或尺度相关性证明检测性能提升。

### 7.3 暴露出的主要问题

- Scratch RT-DETR-L 在当前学习率下波动较大，单个 seed 可能产生明显差异。
- 日志提示部分 CUDA `grid_sampler` 反向操作没有完全确定性实现，因此 `deterministic=True` 不能替代多 seed 实验。
- BTD-SE 在第 57 轮前后切换到稳定的 FP32 辅助损失实现。最严格的最终论文需要从第 1 轮使用稳定实现重新训练。
- BTD-SE 和 VSF-RMR 都出现中期达到峰值、后期明显退化的现象。
- IOQC-SA 的 Precision 略升而 Recall 下降，查询竞争约束可能过强。
- VSF-RMR 的路由缩放幅度增长较大，可能过度修改原始多尺度特征。

## 9. 下一阶段目标

下一阶段不再盲目重复全部 100 轮，而是先回答两个问题：

1. 三个模块是否在各自针对的子场景中产生真实收益？
2. 负增益来自科学假设错误，还是辅助约束和调制强度过大？

### 9.1 第一优先级：补充针对性评估

使用现有最佳权重，在同一验证集上补充：

- BTD-SE：AP-tiny、AP-small、复杂背景误检率。
- IOQC-SA：高密度图像子集 AP、Recall、重复预测率。
- VSF-RMR：尺寸分档 AP、缩放和透视扰动稳定性。

如果某个模块只在目标子场景有效，应调整论文主张，而不是继续声称整体场景普遍提升。

### 9.2 第二优先级：低成本稳定性筛选

所有候选方案从同一个 baseline checkpoint 开始，增加一个不含创新模块、训练轮数相同的 control。先进行短周期配对筛选，只有超过 control 的设置才进入新的 100 轮正式实验。

建议筛选方向：

- IOQC-SA：降低 competition 和 alignment 权重，减少对正确查询的压制。
- BTD-SE：从第 1 轮固定 FP32 辅助损失，降低辅助权重并限制残差调制幅度。
- VSF-RMR：降低尺度辅助权重，并对跨尺度残差幅度设置明确上限。

### 9.3 第三优先级：正式复现实验

当某个候选在配对筛选中稳定超过 control 后，再执行：

1. 固定最终代码和全部参数。
2. 从零训练 100 轮。
3. baseline 与最终方法至少运行多个 seed。
4. 报告均值和标准差。
5. 完成模块消融和复杂度测量。
6. 最后才尝试组合多个创新模块。

## 10. 论文层面的现实策略

论文不需要为了数量强行保留三个无正收益模块。更合理的目标是：

- 一个具有明确科学问题、稳定正增益和完整消融的核心创新。
- 一个能够补充核心方法、且在整体或针对性指标上有正收益的辅助创新。
- 第三个模块只有在实验证明有效后才进入最终论文。

目前优先级为：

1. IOQC-SA：距离 baseline 最近，最值得优先调优。
2. BTD-SE：问题定义和机制较完整，但需要干净复现和稳定化。
3. VSF-RMR：尺度场已学到目标关系，但检测调制过强，需要重新约束。

## 11. 后续实验的禁止事项

- 不再使用动态 batch 结果作为正式对比。
- 不再混用 AdamW、MuSGD、AMP 开关或不同学习率结果。
- 不在训练中途修改损失实现后继续把整条轨迹当作最终唯一结果。
- 不把某个模型的 best.pt 与另一个模型的 last.pt 比较。
- 不在没有 matched baseline 的情况下计算论文提升。
- 不把不同服务器造成的随机差异解释成模块贡献。
- 不在单个独立模块为负增益时直接堆叠三个模块并宣称组合创新。
- 不根据辅助损失下降、注意力图或尺度相关性单独证明检测提升。

我们已经完成了从科学问题定义、三项模块设计、代码实现、稳定训练系统到首轮固定协议实验的完整研究闭环；当前任务已经从“把代码跑起来”转变为“通过针对性评估和低成本配对实验找出真正有效的机制，并形成可重复超过 matched RT-DETR-L baseline 的最终论文方法”。
